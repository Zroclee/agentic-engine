# Agentic Engine 项目说明文档

## 1. 项目架构概述
本项目采用 **Monorepo** 架构进行代码管理，包管理器使用 `pnpm`。项目深度集成了 **Turborepo** (`turbo`) 作为构建系统，以实现极致的构建缓存和任务并行执行。通过 Monorepo 能够将应用（Apps）和公共库（Packages）集中在一个代码库中，方便跨项目共享代码、统筹依赖管理并简化 CI/CD 流程。

项目包含前端 Web 应用（聊天端、管理端）、后端网关（Gateway）以及核心功能库和 UI 组件库。

## 2. 项目目录架构
```text
agentic-engine/
├── apps/                 # 应用程序目录
│   ├── gateway/          # API 网关服务 (后端)
│   ├── chat-web/         # 聊天客户端 Web 应用 (前端)
│   └── admin-web/        # 后台管理端 Web 应用 (前端)
├── packages/             # 公共库与组件目录
│   ├── core/             # 核心库 (业务逻辑、类型定义、Agent 智能体构建)
│   └── ui/               # 公共 UI 组件库 (提供可复用的前端组件与样式)
├── docs/                 # 项目文档
├── .trae/                # Trae IDE 配置与规则
├── pnpm-workspace.yaml   # pnpm monorepo 工作区配置
├── turbo.json            # Turborepo 构建任务配置
└── package.json          # 根目录依赖与脚本
```

## 3. 技术栈与关键库说明

### 3.1 后端服务：Gateway (`apps/gateway`)
- **核心框架**: NestJS (Node.js 框架，基于 TypeScript)
- **主要职责**: 提供 API 接口、路由分发、认证鉴权、微服务/大模型接口调度等。
- **关键依赖**: `@nestjs/core`, `@nestjs/common`, `rxjs`, `reflect-metadata`。

### 3.2 前端应用：Chat Web (`apps/chat-web`) & Admin Web (`apps/admin-web`)
- **核心框架**: Vue 3 (Composition API, `<script setup>`)
- **构建工具**: Vite
- **开发语言**: TypeScript
- **样式与 UI 库**: 
  - **Tailwind CSS**: 实用优先的 CSS 框架，用于快速构建自定义设计。*(后续所有 Web 项目必须引入)*
  - **DaisyUI**: 基于 Tailwind CSS 的组件库，提供语义化的组件类名。*(后续所有 Web 项目必须引入)*
- **主要职责**: 分别面向最终用户和管理员提供现代化的 Web 交互界面。

### 3.3 共享库：Core (`packages/core`)
- **核心技术**: TypeScript, tsup (基于 esbuild 的极速打包工具)
- **主要职责**: 封装核心业务逻辑、定义全局 TS 类型、提供 Agent 智能体构建工具等。作为核心项目，它支持输出 CommonJS 和 ES Modules 双格式，以便于同时被后端 (NestJS) 和前端 (Vue) 引用。

### 3.4 共享库：UI Kit (`packages/ui`)
- **核心技术**: Vue 3, TypeScript
- **主要职责**: 封装企业级或项目级的复用 UI 组件、主题样式、公共资源等。供 `chat-web` 和 `admin-web` 统一引入使用，保证 UI 风格一致性。

## 4. 后续规划
