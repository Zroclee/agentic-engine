以下是针对 AaaS 智能体中台的架构与数据模型设计方案：

**核心数据模型设计**

本次新增的表结构遵循了多租户与项目隔离的原则，关键模型如下：
- **Agent（智能体表）**：归属于特定的 Project。存储核心大模型配置（如 `modelProvider`、`modelName`、`temperature` 等）以及系统提示词（`systemPrompt`）。
- **Tool（工具表）**：归属于特定的 Project。存储 HTTP API 等外部调用的配置定义，通过 JSON 格式 (`apiConfig`) 动态描述。
- **AgentTool（智能体-工具关联表）**：多对多关系。一个智能体可以绑定多个工具，一个工具也可以被多个智能体复用。
- **Session（会话表）**：记录对话。关键设计是增加了 `type` 字段（`DEBUG` / `PRODUCTION`），用于隔离管理员在后端的“调试对话”与最终终端用户在前端的“生产对话”。同时预留了 `endUserId` 字段用于对接外部业务系统的用户 ID。
- **Message（消息表）**：存储具体的上下文。除了 `content` 和 `role` 外，特别增加了 `metaData` 字段用于存储工具调用参数、右侧动态组件联动数据等 AaaS 核心的结构化元数据。

**智能体模块运行时架构设计**

根据项目 README 中的技术栈（NestJS + Python FastAPI），架构需进行**职责分离**：

1. **配置与鉴权层 (Gateway - NestJS)**
   - 负责提供 CRUD API 给前端（如创建智能体、配置提示词、绑定工具）。
   - 负责所有的权限校验（JWT / API Key 等）和数据库落盘操作（保存 Session 和 Message）。
2. **AI 执行层 (AI Service - Python)**
   - 使用 LangChain 或 LlamaIndex 承接实际的模型推理。
   - 网关 (NestJS) 将智能体配置、绑定的工具定义以及对话历史发送给 Python 服务。
   - Python 服务解析意图，执行具体的外部工具调用（HTTP 请求等），并将响应流（SSE 格式）吐回给网关。
3. **前端渲染层 (Web - Vue 3)**
   - 建立 WebSocket 或 SSE 连接监听 NestJS 网关转发回来的大模型流。
   - 解析流中的 `metaData` 消息指令，触发右侧业务组件的动态更新与实时对比。

**对话全链路流程建议**

1. **发起对话**：用户在工作台输入文本，前端通过 REST/WS 发送给 NestJS。
2. **落盘与组装**：NestJS 将用户输入存入 `Message` 表，并提取对应 `Agent` 的 `systemPrompt`、`Tools` 列表和最近 N 条对话记录。
3. **透传至 AI**：NestJS 通过内部 HTTP 调用（或 gRPC）将上述 Payload 发送给 Python AI 服务。
4. **流式响应**：Python AI 结合 LangChain 进行大模型调用，若触发工具调用则直接在 Python 端执行（或返回指令交由 NestJS 执行），最终将文本和组件控制指令（JSON）通过 Streaming 响应给 NestJS。
5. **分发与保存**：NestJS 一边将流转发给前端，一边在流结束时将完整的 Assistant 回复和组件数据落库到 `Message` 表中。
